# Phase 05.1 Plan 01: Design System Tokens and Sidebar Shell Summary

Brief design system tokens (milk/espresso/beeswax palette) added to Tailwind via @theme inline, Playfair Display + DM Sans loaded, two-column espresso sidebar shell replaces minimal dashboard wrapper.

## What Was Built

### Design System Tokens (globals.css)

Added 8 color tokens inside the existing `@theme inline` block alongside shadcn tokens:

```css
--color-milk: #FDF9F0;
--color-espresso: #1C0F05;
--color-beeswax: #C1522A;
--color-beeswax-deep: #A33E1A;
--color-warm-brown: #5C3D1E;
--color-brief-muted: #7A5C3A;
--color-steam: #F0E8D5;
--color-cream: #FAF4E8;
```

And 2 editorial font variables:

```css
--font-playfair: var(--font-playfair-display);
--font-dm-sans: var(--font-dm-sans);
```

### Font Loading (layout.tsx)

- `Playfair_Display` and `DM_Sans` imported from `next/font/google` alongside Geist
- Both exposed as CSS variables on `<body>` via `variable` config option
- Metadata updated: title "Brief", description "Your daily personalised briefing."

### DashboardSidebar Component (src/components/dashboard/sidebar.tsx)

- `"use client"` — requires `usePathname()` for active route detection
- Props: `{ userEmail: string }`
- Structure:
  - Espresso (`bg-espresso`) sidebar, `w-64 min-h-screen`
  - "Brief." logo in Playfair Display 900 weight, milk text, beeswax-deep period
  - Nav: Briefings (`/dashboard`) + Settings (`/dashboard/settings`) — active route gets `border-l-2 border-beeswax text-beeswax`, inactive gets `text-brief-muted hover:text-milk`
  - Bottom: user email (truncated, text-brief-muted) + sign-out form with `signOutAction`

### Dashboard Layout Update (dashboard/layout.tsx)

Replaced minimal `min-h-screen bg-background` wrapper with:
- Two-column flex: `DashboardSidebar` + `<main className="flex-1 bg-milk relative overflow-auto">`
- Fixed SVG grain texture overlay (`feTurbulence fractalNoise`) at `z-0` with `opacity: 0.4`
- Content wrapped in `<div className="relative z-10">` to render above grain

### Dashboard Page Cleanup (dashboard/page.tsx)

- Removed duplicate `<form action={signOutAction}>` and `Button` import
- Sidebar now handles all sign-out; page is a clean placeholder with session email display

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| `--color-brief-muted` for #7A5C3A | Avoids shadowing shadcn's `--color-muted` (gray); Brief tokens must coexist cleanly |
| Tokens inside existing `@theme inline` block | One `@theme` block per CSS file — creating a second would conflict |
| DashboardSidebar as Client Component | `usePathname()` requires browser runtime; server layout stays Server Component |
| SVG grain overlay | No external image assets; pure CSS noise renders inline without network round-trips |
