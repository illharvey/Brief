Try again.")

## Decisions Made
- `saveSettingsAction` is a plain async function (not a server action) in the Client Component — calls saveTopicsAction then saveDeliveryPreferenceAction, short-circuits on topics error. This composes two existing server actions without modifying them.
- Timezone re-detected client-side every render via `Intl.DateTimeFormat().resolvedOptions().timeZone` — initialTimezone prop from server is accepted but not used for state initialization; fresh detection is more accurate.
- Toaster placed in root layout body — not dashboard layout — so it's available across all routes.
- TIME_SLOTS has 17 entries: plan CONTEXT mentioned "16 options" but 6 AM through 10 PM at 1-hour increments produces 17 values; the plan interfaces section explicitly notes this and says implement 17.
