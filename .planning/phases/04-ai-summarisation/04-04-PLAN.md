<task type="checkpoint:human-verify">
  <name>End-to-end verification: run pipeline, inspect output, audit grounding</name>
  <what-built>
    Complete summarisation pipeline: generateBriefingForUser() orchestrates article fetch → ranking → LLM summarisation with Redis cache → markdown assembly → DB persist. Exposed via POST /api/dev/summarise and scripts/generate-briefing.ts.
  </what-built>
  <how-to-verify>
    **Pre-requisites:** Ensure the following are set in `.env.local`:
    - `ANTHROPIC_API_KEY` — from Anthropic Console → API Keys
    - `SUMMARISATION_MODEL` — optional (defaults to `claude-haiku-4-5-20251001`)
    - `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN` — existing from Phase 1
    - `DATABASE_URL` — existing from Phase 1

    Also ensure Phase 3 ingestion has been run first so articles exist in the DB. If not, trigger ingest first: `POST /api/dev/ingest` with `{ "userId": "<your-user-id>" }`.

    **Step 1 — Run the dev server (if not already running):**
    ```bash
    npm run dev
    ```

    **Step 2 — Get your userId from the database or Neon console, then run the pipeline:**

    Option A — via curl:
    ```bash
    curl -X POST http://localhost:3000/api/dev/summarise \
      -H "Content-Type: application/json" \
      -d '{"userId": "<your-user-id>"}'
    ```

    Option B — via CLI script:
    ```bash
    npx tsx scripts/generate-briefing.ts <your-user-id>
    ```

    **Expected Step 2 output:** JSON (or console output) containing:
    - `briefingId`: a UUID
    - `topicCount`: number > 0
    - `itemCount`: number > 0
    - `content`: markdown string with `## TopicName` headers and `- bullet [Source](url)` lines
    - `partialFailure`: false (or true with `failedTopics` list if Anthropic API has issues)

    **Step 3 — Verify cache behaviour (no duplicate LLM calls):**
    Run the pipeline a second time for the same user:
    ```bash
    curl -X POST http://localhost:3000/api/dev/summarise \
      -H "Content-Type: application/json" \
      -d '{"userId": "<your-user-id>"}'
    ```
    Expected: Response is fast (< 2 seconds vs. 10-30 seconds first run). The Anthropic Console usage page should show no new tokens billed for the second run.

    **Step 4 — Manual grounding audit (10+ bullets):**
    From the `content` field, identify 10 or more bullet points. For each:
    1. Note the source name in the `[Source Name](url)` attribution
    2. Open the source URL
    3. Confirm the claim in the bullet is supported by the article text

    OR: Query the DB for `briefing_items` rows with `briefing_id = <briefingId>` and inspect the `source_snapshot` column. Every claim in `summary` should be traceable to text in `source_snapshot`.

    **Step 5 — Confirm DB persistence:**
    Check the Neon console (or run a DB query) to confirm:
    - A row exists in `briefings` with the returned `briefingId`
    - Rows exist in `briefing_items` with `briefing_id = <briefingId>`

    **Known acceptable failures:**
    - `partialFailure: true` with Guardian API in failedTopics — expected (requires paid plan)
    - Reddit topics failing — expected (Reddit blocks scrapers)
    - Zero items for a topic — acceptable if no articles were ingested for that topic today
  </how-to-verify>
  <resume-signal>
    Type "verified" if the pipeline produces a valid briefing with grounded bullets and cache works correctly.
    Or describe any issues found (e.g. "bullets truncated mid-sentence", "API key error", "zero items").
  </resume-signal>
</task>
