<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure mail.brief.app in Resend dashboard and add DNS records</name>
  <action>
This task requires human action in external dashboards — Claude cannot access Resend or DNS providers.

**Step 1 — Add domain in Resend dashboard:**
1. Go to https://resend.com/domains
2. Click "Add Domain"
3. Enter: `mail.brief.app` (the subdomain — NOT `brief.app` root; Phase 1 sends from `noreply@mail.brief.app`)
4. Resend will display DNS records to add. Copy all of them.

**Step 2 — Add DNS records at your DNS provider:**

Add these records for `mail.brief.app`. Exact values come from the Resend dashboard — copy them verbatim:

| Type | Name/Host | Value |
|------|-----------|-------|
| TXT | `mail.brief.app` | (Resend-generated SPF record) |
| TXT | `resend._domainkey.mail.brief.app` | (Resend-generated DKIM key; selector name may differ slightly) |
| MX | `mail.brief.app` | (Resend-generated MX value — enables bounce routing) |
| TXT | `_dmarc.mail.brief.app` | `v=DMARC1; p=none; rua=mailto:dmarc-reports@brief.app` |

The DMARC record is added manually — Resend does not generate it. `p=none` is correct for Phase 2 (monitoring only; satisfies Gmail/Yahoo/Microsoft 2025 requirements).

**Step 3 — Create Resend webhook endpoint:**
1. Go to https://resend.com/webhooks
2. Click "Add Endpoint"
3. URL: `https://<your-deployed-app-url>/api/webhooks/resend`
4. Events: select `email.bounced` and `email.complained`
5. Copy the Signing Secret — this becomes `RESEND_WEBHOOK_SECRET`

**Step 4 — Add environment variables:**

In `.env.local`:
```
RESEND_WEBHOOK_SECRET=<paste Signing Secret from Step 3>
UNSUBSCRIBE_SECRET=<generate: openssl rand -hex 32>
```

In Vercel (or your deployment platform) → Environment Variables, add the same two variables.
