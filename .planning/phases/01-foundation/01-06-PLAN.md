  <what-built>
    Complete Phase 1 foundation: Next.js 15 project with Auth.js v5 (Credentials provider, JWT sessions, split-config for Edge middleware), Drizzle ORM on Neon Postgres (7 tables including custom GDPR consent, password reset tokens, user topics, and delivery preferences), rate-limited login, 3-step onboarding flow, password reset flow, email verification, and protected dashboard.

    Before running verification, ensure:
    1. .env.local has real values for: DATABASE_URL (Neon), AUTH_SECRET (generated), RESEND_API_KEY (Resend dashboard), UPSTASH_REDIS_REST_URL and UPSTASH_REDIS_REST_TOKEN (Upstash console), APP_URL=http://localhost:3000
    2. Run `npx drizzle-kit push` to create all tables in Neon
    3. Run `npm run dev` — server starts on http://localhost:3000
  </what-built>
  <how-to-verify>
    Run through each flow in this order. Use a real email address you can check for verification emails.

    **Flow 1: Full signup (AUTH-01, AUTH-06, PREF-01, PREF-02)**
    1. Visit http://localhost:3000/signup
    2. Enter email and a valid password (8+ chars, 1 uppercase, 1 number) → click Continue
    3. On topics page: type "Formula 1", press Enter; type "AI Policy", press Enter; type "UK Politics", press Enter → click Continue
    4. On delivery page: confirm timezone is auto-detected (shows your IANA timezone); change time to "08:00" → click Finish setup
    5. Confirm page shows — should read "You're all set" with verification email prompt
    6. Check your email — verify the verification email arrived from noreply@mail.brief.app
    7. Click the verification link in the email → should reach /verify-email with success message

    **Flow 2: Session persistence (AUTH-02)**
    8. Visit http://localhost:3000/login — log in with the credentials from Step 2
    9. You should reach /dashboard showing your email
    10. Refresh the page — session should persist, /dashboard still shows your email without redirecting to login

    **Flow 3: Sign out (AUTH-05)**
    11. On /dashboard, click Sign out
    12. You should be redirected to /login
    13. Visit http://localhost:3000/dashboard directly — you should be redirected back to /login (middleware protecting the route)

    **Flow 4: Duplicate email error**
    14. Visit http://localhost:3000/signup and attempt to sign up with the same email from Step 2
    15. Should show "An account with this email exists — log in?" with a link to /login

    **Flow 5: Rate limiting (AUTH-03 error handling)**
    16. Visit http://localhost:3000/login
    17. Enter correct email but wrong password 5 times quickly
    18. Should show "Too many attempts. Try again in X seconds." message
    19. Wait 30 seconds, try again — should work normally

    **Flow 6: Password reset (AUTH-04)**
    20. Visit http://localhost:3000/forgot-password
    21. Enter your email → click Send reset link
    22. Page should show "Check your inbox" message
    23. Check your email — a password reset email should arrive
    24. Click the reset link → should reach /reset-password?token=...
    25. Enter a new valid password and confirm → click Set new password
    26. Should redirect to /login
    27. Log in with the new password — should work; log in with the old password — should fail

    **Flow 7: Database verification**
    28. In Neon console (https://console.neon.tech), open your database and run:
        ```sql
        SELECT * FROM users WHERE email = 'your-test-email';
        SELECT * FROM user_consents WHERE user_id = '{your user id}';
        SELECT * FROM user_topics WHERE user_id = '{your user id}';
        SELECT * FROM delivery_preferences WHERE user_id = '{your user id}';
        ```
    29. Confirm: users row has email_verified timestamp (after Flow 1 verification); user_consents has 2 rows (marketing_email + terms_of_service) with real IP address; user_topics has 3 rows (Formula 1, AI Policy, UK Politics); delivery_preferences has 1 row with delivery_time "08:00" and your IANA timezone

    **Expected results:**
    - All 7 flows complete without errors
    - Emails arrive from Resend (may take 1-2 minutes)
    - Database shows all expected rows with correct data
    - Email verification link marks emailVerified timestamp
    - Password reset link cannot be reused (try clicking it a second time — should show "expired or already been used")
